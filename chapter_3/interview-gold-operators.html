PART 1️⃣ â€” & vs && (AND Operators)

1. What is & ?

& is a bitwise AND operator
But when used with booleans, it works as a logical AND without short-circuiting.

👉 Both operands are ALWAYS evaluated

int a = 5, b = 3;
System.out.println(a & b);   // Bitwise AND â†’ 1
    
boolean x = false;
boolean y = (10 / 0 > 1);  // ERROR
System.out.println(x & y); // ❌ Runtime error (division by zero)

2. What is && ?

&& is a logical AND with short-circuiting

👉 Second condition executes ONLY if first is true

boolean x = false;
boolean y = (10 / 0 > 1);
System.out.println(x && y); // ✅ No error (y not evaluated)

4. Real Interview Example 

if(user != null & user.isActive())  // ❌ Risk
    
if(user != null && user.isActive()) // ✅ Correct

📌 Why?
If user is null, && stops execution â†’ no NullPointerException.

5. Interview Question ⭐ ⭐ ⭐

Q: Why is && preferred over & in conditions?
A: Because && uses short-circuit evaluation, improving performance and safety by preventing unnecessary execution.

PART 2️⃣ â€” Short-Circuit Operator (Deep Understanding)

Definition
Short-circuit means:

> If result is already known, remaining conditions are skipped 

false && expensiveFunction(); // expensiveFunction NOT called
true || expensiveFunction();  // expensiveFunction NOT called

Operators that short-circuit:
✅ &&
✅ ||

Operators that DO NOT:
❌ &
❌ |

PART 3️⃣ â€” Ternary Operator (?:)

Syntax

condition ? value_if_true : value_if_false;

Example

int max = (a > b) ? a : b;

Equivalent if-else:

int max;
if(a > b) {
    max = a;
} else {
    max = b;
}
