1. What is a switch statement?

A switch statement is a multi-branch decision-making control statement. It lets you execute one block of code out of many, based on the value of an expression.

üëâ It is an alternative to long if-else if ladders when you compare one variable with many fixed values.

2️⃣ Syntax - OLD switch (Traditional)

switch (expression) {
    case value1:
        // code
        break;
    case value2:
        // code
        break;
    default:
        // code
}

3. Rules of OLD switch

Expression rules

  • byte
  • short
  • char
  • int
  • enum
  • String (since Java 7)

❌ Not allowed:

  • float, double
  • long
  • boolean

Case rules

  • case values must be constant
  • case values must be unique
  • Order does not matter
  • default is optional

4. Example 1 - Simple Integer Switch

int day = 3;

switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    default:
        System.out.println("Invalid day");
}

Explanation

  • day = 3
  • Control jumps to case 3
  • Prints Wednesday
  • break stops execution

5. Why break is IMPORTANT?

Without break, fall-through happens.

int num = 2;

switch (num) {
    case 1:
        System.out.println("One");
    case 2:
        System.out.println("Two");
    case 3:
        System.out.println("Three");
}

Output

Two
Three

Why?
case 2 matches, no break, execution continues to next cases.

6. Intentional Fall-through (Real Use)

int month = 2;

switch (month) {
    case 12:
    case 1:
    case 2:
        System.out.println("Winter");
        break;
    case 3:
    case 4:
    case 5:
        System.out.println("Summer");
        break;
}

‚úî Used when multiple cases need same logic

10. Problems with OLD switch

  • Too many break
  • Easy to forget break
  • More lines of code
  • Error-prone

11. New Switch Expression (Arrow ‚Üí)

switch (expression) {
    case value -> statement;
    case value2 -> statement;
    default -> statement;
}

12. Example - New Switch

int day = 1;

switch (day) {
    case 1 -> System.out.println("Monday");
    case 2 -> System.out.println("Tuesday");
    default -> System.out.println("Invalid day");
}
  • No break needed
  • No fall-through
  • More readable

13. New Switch with Multiple Values

int month = 2;

switch (month) {
    case 12, 1, 2 -> System.out.println("Winter");
    case 3, 4, 5 -> System.out.println("Summer");
    default -> System.out.println("Invalid month");
}

14. Switch as an Expression (Returns Value)

int day = 3;

String result = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Invalid day";
};

System.out.println(result);

15. Using yield (Block Switch)

int num = 10;

String type = switch (num) {
    case 0 -> "Zero";
    case 1 -> "One";
    default -> {
        if (num > 0)
            yield "Positive number";
        else
            yield "Negative number";
    }
};

System.out.println(type);

16. Old vs New Switch - Comparison

Feature Old Switch New Switch
Break required Yes No
Fall-through Yes No
Returns value No Yes
Readability Medium High
Error-prone More Less

17. When to Use Switch?

‚úÖ Use switch when:

  • Comparing one variable
  • With fixed values
  • Better readability than if-else

‚ùå Avoid switch when:

  • Complex conditions
  • Ranges (>, <)
  • Boolean logic
