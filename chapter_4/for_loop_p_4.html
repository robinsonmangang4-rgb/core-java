1. What are Control Statements?

Control statements decide how your program flows:

  • What runs
  • How many times it runs
  • When it stops

Java control statements:

  • Selection →  if ,  switch 
  • Iteration → for ,  while ,  do-while 
  • Jump →  break , continue ,  return 

✅We are Focusing on Iteration →  for  loop

2. What is a  for  Loop? ( Beginner Level )

A  for loop is used when:

  • you know how many times you want to repeat something.

    • Real-life example

    • "Print attendance for 30 students "
    • "Run a task 10 times"

    That's a  for  loop job.

    3. Basic Syntax (Must Memorize)

    for (initialization; condition; update) {
  // body
}

    Breakdown ( IMPORTANT )

    Part Meaning
    initialization Runs Once
    condition Checked before every loop
    update Runs after each Iteration
    body Code that repeats

    4. First  for  Loop Program ( Zero level ) 

    public class Main {
  public static void main(String[] args) {
      for (int i = 1; i <= 5; i++) {
          System.out.println(i);
      }
  }
}

    Execution Flow ( Step-by-step )

    Step Value of  i  Condition Output
    1 1 true 1
    2 2 true 2
    3 3 true 3
    4 4 true 4
    5 5 true 5
    6 6 false loop stops

    Golden rule:

    If condition becomes  false , loop ends.

    5. Common Beginner Mistakes

    ✗ Infinite Loop

    for (int i = 1; i <= 5; ) {
  System.out.println(i);
}

    👉 No update → infinite loop

    ✗ Wrong Condition

    for (int i = 5; i >= 1; i++) {
  System.out.println(i);
}

    👉  i++  increases forever → infinite loop

    6. Reverse Loop ( Basic but Powerful )

    for (int i = 10; i >= 1; i--) {
  System.out.println(i);
}

    📌 Used in:

    • Countdown timers
    • Stack processing
    • Reverse arrays

    7. Multiple Statements in  for  Loop

    Multiple variables

    for ( int i = 1, j = 10; i <= 5; i++, j--) {
    System.out.println ( i + " " + j);
}

    ✅ Allowed
    ❌ Different data types not allowed in initialization

    8.  for  Loop Without Curly Braces ( ⚠ Dangerous)

    for ( int i = 1, i <= 3; i++,) {
    System.out.println ( i );
    System.out.println ( " Hello" );
}

    👉Only first statement is inside Loop
    Always use {}

    9. Nested  for  Loops ( Intermediate Level )

    Pattern Printing (Classic Interview Topic)

    for (int i = 1; i <= 3; i++) {
  for (int j = 1; j <= 3; j++) {
      System.out.print("* ");
  }
  System.out.println();
}

    Output

    * * *
* * *
* * *

    How it works

    • Outer loop → rows
    • Inner loop → columns

    10. Triangle Pattern Example (Interview Favorite)

    for (int i = 1; i <= 5; i++) {
  for (int j = 1; j <= i; j++) {
      System.out.print("* ");
  }
  System.out.println();
}

    Output

    * 
* * 
* * *
* * * *
* * * * *

    11.  for  Loop with  break 

    for (int i = 1; i <= 5; i++) {
  if ( i == 5) {
    break;
  }
  System.out.println(i);
}

    Output

    1 2 3 4

    👉  break  exits loop immediately

    12.  for  Loop with  continue 

    for (int i = 1; i <= 5; i++) {
  if ( i == 3) {
    break;
  }
  System.out.println(i);
}

    Output

    1 2 4 5

    👉  continue  skips current iteration only

    13. Advanced: Enhanced  for  Loop ( for-each )

    Used for:

    • Arrays
    • Collections

    Syntax

    for (datatype variable : array){
    // code
}

    Example

    int[] numbers = {10, 20, 30};

for (int num : numbers) {
  System.out.println(num);
}
    • 📌 Limitation
    • ❌ index access
    • ❌ Cannot modify structure

    14. Advanced:  for(;;)  Infinite Loop

    for (;;){
    System.out.println("Running forever");
}

    ✅ Valid Java
    Used in :

    • Game loops
    • Servers
    • Embedded systems

    15. Scope of Variables in  for  Loop (Tricky)

    for (int i = 1; i <= 3; i++){
    System.out.println("i");
}
// System.out.println(i); ❌ ERROR

    👉  i  exists only inside the loop

    16. Performance & Best Practices (Pro Level)

    ❌Bad

    for (int i = 0; i < arr.length; i++) {
  }

    ✅ Better

    int len = arr.length;
for (int i =0; i< len; i++){
}

    👉 Reduces repeated property access (minor but pro habit)

    17. Interview Questions ( Must Know )

    Q1. Can condition be omitted?

    ✅ Yes 

    for (int i = 1;; i++){
    System.out.println(i);
}

    Q2. Is  for  loop faster than  while ?

    ❌ No real difference

    ✅ Choice depends on readability


    Q3. Can we write logic in update part?

    ✅ Yes

    for (int i = 1; i<= 10;  i += 2){
     System.out.println(i);
}

    Real-world Use Cases

    • Iterating database results
    • Processing arrays & collections
    • Generating reports
    • Pattern & algorithm problems
    • Game engines & animationss

    Pro Tip Summary

    • ✔ Use for loop when count is known
    • ✔ Prefer enhanced for for read-only data
    • ✔ Avoid infinite loops unless intentional
    • ✔ Always use curly braces
    • ✔ Master nested loops for interviews
