Interview-Ready Explanation: Java Core Concepts

Below is a complete, interview-ready explanation of Literals, Variable Scope, the final keyword, and why char uses 2 bytes in Java, with clear theory + examples.

1. Literals in Java

📌 Definition

Literals are fixed constant values directly written in the source code.

📌 Types of Literals

1. Integer Literals

int a = 10;      // decimal
int b = 012;     // octal
int c = 0xA;     // hexadecimal
int d = 0b1010;  // binary (Java 7+)

2. Floating-Point Literals

float f = 10.5f;
double d = 20.5;

3. Character Literals

char ch1 = 'A';
char ch2 = 65;       // ASCII value
char ch3 = '\u0041'; // Unicode for 'A'

4. String Literals

String s = "Hello";

5. Boolean Literals

boolean isJavaFun = true;

6. Null Literal

String s = null;

📌 Interview Tip:

> String literals are stored in the String Constant Pool for memory optimization.

2. Variable Scope in Java

📌 Definition

Scope defines where a variable is accessible in a program.

1. Local Variables

  • Declared inside methods or blocks
  • No default value
  • Must be initialized before use
void show() {
    int x = 10;
    System.out.println(x);
}

2. Instance Variables

  • Declared inside a class but outside methods
  • Each object gets its own copy
  • Have default values
class Test {
    int a; // default value = 0
}

3. Static Variables

  • Declared using static
  • Shared among all objects
  • Loaded when class is loaded
class Test {
    static int count = 0;
}

4. Block Scope

if (true) {
    int x = 5;
}
// x not accessible here

📌 Interview Tip:

> Java does not support global variables.

3. final Keyword in Java

The final keyword is used to restrict modification.

1. final Variable

Value cannot be changed

final int MAX = 100;

📌 Blank final variable:

final int x;
x = 10; // allowed once

2. final Method

Cannot be overridden

class Parent {
    final void show() {
        System.out.println("Final Method");
    }
}

3. final Class

Cannot be inherited

final class Test {
}

📌 Interview Tip:

> final improves security, immutability, and performance.

4. Interview Question: Why does char use 2 bytes in Java?

✅ Short Answer

Because Java uses Unicode, not ASCII.

📌 Detailed Explanation

Feature Explanation
Size of char 2 bytes (16 bits)
Range 0 to 65535
Encoding Unicode (UTF-16)

📌 Why Unicode?

  • ASCII supports only 128 characters
  • Java was designed for internationalization
  • Unicode supports multiple languages:
    • English
    • Hindi
    • Chinese
    • Arabic
    • Symbols & Emojis

All characters occupy 2 bytes.

  • Java char represents one UTF-16 code unit
  • Some emojis require two chars (surrogate pairs)

> Java uses 2 bytes for char because it follows Unicode (UTF-16) to support international characters, unlike ASCII which uses 1 byte.
