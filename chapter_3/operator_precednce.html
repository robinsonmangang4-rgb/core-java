1. What is Operator Precedence?

Operator precedence decides which operator is evaluated first in an expression when multiple operators are present. 

2 + 3 * 4
Result: 14

2. Why Operator Precedence Matters?

  • You may get wrong output
  • Code may work incorrectly
  • Interviews love to test this 🙂
int result = 10 + 5 * 2;
System.out.println(result); // 20, not 30

4. Arithmetic Operator Precedence

int result = 10 + 5 * 2 - 4 / 2;
System.out.println(result); // 18

5. Parentheses Override Precedence

int result = (10 + 5) * 2;
System.out.println(result); // 30

6. Unary Operator Precedence (++ / --)

int a = 5;
int b = ++a;
System.out.println(a); // 6
System.out.println(b); // 6
    
int a = 5;
int b = a++;
System.out.println(a); // 6
System.out.println(b); // 5

7. Relational vs Logical Precedence

int a = 10, b = 20;
System.out.println(a < b && b > 15); // true

8. Bitwise vs Logical Operators

int a = 5, b = 3;
System.out.println(a & b + 1); // 4

9. Ternary Operator Precedence

int a = 10, b = 20;
int max = a > b ? a : b;
System.out.println(max); // 20

10. Assignment Operator (Lowest Precedence)

int a = 10;
a += 5 * 2;
System.out.println(a); // 20

11. Associativity

int result = 10 - 5 - 2;
System.out.println(result); // 3
    
int a, b, c;
a = b = c = 10;

13. Best Practices ⭐

  • Use parentheses for readability
  • Avoid complex expressions
  • Break logic into multiple lines
  • Never rely only on precedence in real projects
